# PORTFOLIO DATA EXTRACTION META-PROMPT

## PURPOSE
This prompt guides an LLM to extract structured data from uploaded resume documents and generate JSON files matching the Keystatic CMS schema for Shreyans Bhatt's portfolio website.

## INPUT REQUIREMENTS
- Multiple resume files (PDF, DOCX, ODT, TXT formats)
- LinkedIn profile data (if available)
- Cyron MVP Skills Assessment document

## OUTPUT FORMAT
Generate JSON files matching these exact schemas:

---

## 1. PROFILE SINGLETON (src/content/profile/main.json)

Extract and synthesize:
- `name`: Full legal name
- `title`: Most senior/current role title
- `tagline`: Synthesize a compelling one-liner from professional summary
- `email`: Primary contact email
- `phone`: Contact number (optional)
- `avatarImage`: Path if available, else null
- `location`: Current city, country
- `timezone`: Derive from location (e.g., "UTC+5:30" for India)
- `availabilityStatus`: Default to "Open to Discuss"
- `openTo`: Extract from resume preferences or default to ["Contract", "Full-time", "Advisory"]
- `workModes`: Extract or default to ["Remote", "Hybrid"]
- `socialLinks`: Extract all social URLs
  - linkedin, github, twitter, medium
- `aspirations`: Synthesize 3-5 career goals from experience trajectory
  - Each: { goal, timeline: "Immediate"|"Strategic"|"Visionary", icon }
- `bio`: Extended professional summary in MDX

---

## 2. PROJECTS COLLECTION (src/content/projects/*.json)

For EACH work experience entry, extract:

```json
{
  "slug": "kebab-case-company-name-yyyy",
  "clientName": "Company Name",
  "displayName": null, // or masked name if confidential
  "role": "Job Title",
  "startDate": "YYYY-MM",
  "endDate": "YYYY-MM" or "Present",
  "engagementType": "Permanent" or "Contract",
  "workMode": "Remote" or "Hybrid" or "Onsite",
  "teamSize": number or null,
  "location": "City, Country",
  "isConfidential": false, // true for NDA-bound clients
  "isFeatured": true/false, // mark top 5-6 as featured
  "isDraft": false,
  "companyDescription": "Brief company description",
  "techStack": ["Tech1", "Tech2", ...],
  "skills": [
    { "name": "Skill Name", "rating": 1-10 } // PROJECT-SPECIFIC depth
  ],
  "achievements": ["Achievement 1", "Achievement 2", ...],
  "projectReference": "URL if available"
}
```

### SKILL RATING GUIDELINES (Project-Specific Depth)
- 9-10: Primary technology, architectural decisions, mentoring others
- 7-8: Deep hands-on implementation, complex features
- 5-6: Moderate usage, standard implementations
- 3-4: Supporting technology, light usage
- 1-2: Minimal exposure, awareness level

### UNION RULE
When multiple resumes describe the same project with different details:
- UNION all achievements (deduplicate)
- UNION all tech stack items
- Take the MAXIMUM skill rating for overlapping skills
- Prefer the more detailed description

## 9. VALIDATION CHECKLIST

Before finalizing, verify:
- [ ] All dates are in YYYY-MM format
- [ ] All slugs are kebab-case
- [ ] All skill ratings are 0-10 integers
- [ ] Tech stack items are capitalized consistently
- [ ] No duplicate projects
- [ ] Featured projects are marked (5-6 max)
- [ ] Profile social links are valid URLs
- [ ] Achievements are quantified where possible (%, numbers)

---

## 10. EXECUTION COMMAND

After generating all JSON files, run:
```bash
# Validate JSON syntax
find src/content -name "*.json" -exec jsonlint {} \;

# Start Keystatic admin
npm run keystatic
```

Access admin at http://localhost:4321/keystatic to review and refine.
